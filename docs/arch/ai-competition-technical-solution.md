# 粮达健康 AI 竞赛技术方案

拟采用技术路线：采用“大模型 + 健康报告知识库 + 家庭健康画像引擎 + 推荐 Agent”的核心架构，面向家庭健康管理与智能营销场景。

核心模块与实现路径：输入层采集体检报告、家庭成员档案、近 7 天手环数据、饮食偏好及预算信息。核心层对报告进行 OCR、文本切片、向量化和健康事实抽取，保留页码与原文证据，构建可追溯的健康报告知识库；再结合个人档案、历史记忆和设备数据生成成员级或家庭级健康画像；由推荐 Agent 动态调用报告检索、画像聚合、商品筛选和排序能力。输出层生成健康建议、商品推荐及推荐理由。

实现路径上，系统先完成家庭成员识别与数据隔离，避免多成员信息混用；再按需组织健康事实、长期记忆、商品候选和安全约束；最后完成候选生成、安全过滤、排序和证据链输出，并把点击、跳过、加购等反馈写回记忆系统，持续优化下一轮推荐。

技术难点与应对思路：第一，健康场景下大模型容易出现“幻觉”，需通过 RAG 检索、结构化健康事实和规则引擎做双重约束；第二，家庭多成员场景容易发生画像串用，需按 `member_id` 严格隔离报告、记忆和推荐结果；第三，用户偏好可能与健康约束冲突，需设定“过敏禁忌 > 报告事实 > 健康标签 > 记忆偏好”的优先级，保证推荐安全可控。
