# 粮达健康 - 长期市场化迭代规划

日期：2026-06-22
周期：1-3年
适用范围：从最小可用demo走向市场化家庭健康智能营销平台的演进蓝图
目标读者：产品 / 架构 / 业务 / 集团对接团队

---

## 0. 写在前面

当前 demo 已经验证了一条最小闭环：

```
上传报告 → 健康事实 → 健康画像 → 餐单 → 商品 → 反馈
```

长期规划的核心使命是让这条闭环从"单次演示"变成"持续运营的家庭健康数据资产 + 集团消费品精准转化引擎"。

规划的原则：

1. **以数据资产为核心**——所有功能围绕"采集 → 沉淀 → 理解 → 转化 → 反馈"展开，不为技术而技术。
2. **以家庭为单位**——区别于传统个人健康 App，粮达健康始终围绕"家庭健康图谱"做决策。
3. **以集团为底盘**——所有功能必须能挂在中粮集团消费品矩阵（福临门、长城、中茶、悦鲜活、香雪、五谷道场、蒙牛合作产品等）与粮达网供应链上。
4. **以健康安全为底线**——任何个性化都不能突破健康禁忌，记忆和营销不得覆盖健康红线。
5. **以可解释为信任**——所有推荐必须可追溯到健康事实 / 行为记忆 / 商品标签 / 集团商品库。

## 1. 产品愿景

### 1.1 一句话定位

> **粮达健康 = 中粮集团旗下的家庭健康数据资产平台 + 健康驱动消费品精准营销引擎。**

它不是单纯的健康 App，也不是单纯的电商，而是一座连接"家庭健康需求"和"中粮全产品线供给"的智能桥梁。

### 1.2 战略对齐

| 维度 | 对齐 |
| --- | --- |
| 国家战略 | 健康中国 2030、乡村振兴、央企数字化转型 |
| 集团战略 | 中粮集团"从田间到餐桌"全产业链、向消费端延伸 |
| 行业趋势 | 主动健康、家庭健康、银发经济、营养个性化 |
| 技术趋势 | 多模态大模型、Agent 工具调用、IoT 健康设备、隐私计算 |

### 1.3 价值主张

| 主体 | 价值 |
| --- | --- |
| 家庭用户 | 一份报告、一个家庭、一日三餐的健康管家 |
| 中粮集团 | 集团全产品线精准转化的私域引擎 |
| 区域合作伙伴 | 健康消费洞察反哺区域选品与配送 |
| 供应链 | 健康趋势预测反哺种植 / 加工 / 库存 |
| 生态合作方 | 保险、体检、医疗、养老机构的健康数据接口 |

## 3. 数据采集层（L2 上半段）— 家庭健康数据全景化

核心命题：**让家庭健康数据从"用户主动上传一份报告"扩展为"无感、连续、多源的家庭健康数据流"。**

### 3.1 数据来源矩阵

| 来源类型 | 数据项 | 采集方式 | 优先级 |
| --- | --- | --- | --- |
| 健康报告 | 体检报告、化验单、影像报告、出院小结 | PDF 上传 + OCR + LLM 结构化抽取 | P0 |
| 可穿戴设备 | 心率、HRV、血氧、睡眠、步数、压力 | BLE / 云端同步（华为、小米、苹果、OPPO） | P1 |
| 家用健康设备 | 血压、血糖、体温、体重、体脂、尿酸 | 蓝牙 / 4G Cat.1 模组对接 | P1 |
| 厨房智能设备 | 智能冰箱（库存）、智能秤（烹饪习惯）、净水器 | IoT MQTT / 厂商云对接 | P2 |
| 商城行为 | 浏览、加购、下单、复购、收藏、评价 | 埋点 + 商城后台 | P0 |
| 对话行为 | Agent 多轮对话、点赞、点踩、追问 | 对话日志 + 反馈埋点 | P0 |
| 主动录入 | 餐食打卡、自测症状、过敏登记 | 表单 + 拍照 + 语音 | P1 |
| 地理位置 | 家庭 / 公司 / 餐厅常驻区域 | 手机定位授权 | P2 |
| 集团内协同 | 集团会员数据、消费券、线下到店 | 中粮集团会员中台对接 | P1 |
| 第三方授权 | 医院 HIS、体检机构、互联网医院 | API 授权 + 隐私计算 | P3 |

### 3.2 采集层关键能力

1. **设备协议适配网关**——统一抽象 BLE / MQTT / HTTP / SDK 数据流。
2. **数据质量与时序治理**——缺失值、异常值、单位换算、时区对齐。
3. **隐私优先的采集设计**——本地预处理后再上传，最小化上传字段。
4. **集团数据中台对接**——通过中粮数据中台拉取会员、券、订单基础信息。

### 3.3 数据合规与伦理

- 所有健康数据采集需用户明示同意，家庭成员需分别授权。
- 涉及未成年人数据按《个人信息保护法》严格隔离。
- 集团内数据共享走隐私计算或差分隐私，不直接落库。
- 数据保留周期与用户撤回机制明示。

## 4. 数据资产层（L3）— 健康与商业数据沉淀

核心命题：**把零散的数据点，沉淀为可复用的数据资产。**

### 4.1 五大核心数据资产

| 资产 | 作用 | 关键字段 |
| --- | --- | --- |
| **health_facts** 健康事实库 | 长期健康证据 | fact_type、name、value、status、source_doc_id、evidence |
| **behavior_memory** 行为记忆库 | 跨会话偏好与目标 | preference、avoidance、goal、marketing_feedback |
| **family_graph** 家庭健康图谱 | 多成员关系 | member_id、relation、shared_meal、shared_goal |
| **product_tag_index** 商品标签索引 | 集团商品匹配 | sku、tags（低钠/低脂/高钙等）、health_role、family_role |
| **health_timeline** 健康时间线 | 趋势与节点 | member_id、event_type、event_date、related_fact_id |

### 4.2 资产之间的关联

```
health_facts   ──┐
                 ├──> family_health_profile  ──> Agent 推荐
behavior_memory──┤
product_tag    ──┘
       │
       └──> health_timeline  ──> 长期效果追踪 & 集团洞察
```

### 4.3 资产治理

- 统一 ID 体系（member_id / family_id / sku / fact_id / memory_id）。
- 数据血缘（每个 fact / memory 都可追溯到采集源与时间）。
- 冷热分层（活跃家庭热数据，沉默家庭冷数据归档）。
- 用户控制面板（可查看、编辑、删除、导出）。

## 5. 智能决策层（L4）— 从数据到洞察

核心命题：**让决策不止于"今天吃什么"，而是"未来 3 个月如何持续改善家庭健康指标"。**

### 5.1 决策模型矩阵

| 模型 | 输入 | 输出 | 算法 |
| --- | --- | --- | --- |
| 健康画像聚合 | facts + 档案 + 设备 | 长期风险 / 饮食原则 / 禁忌 | 规则 + LLM |
| 健康风险预测 | 时间线 + facts + 设备趋势 | 3/6/12 个月风险评分 | 时序模型 + 规则 |
| 餐单生成 | 画像 + 记忆 + 库存 + 季节 | 一日 / 一周餐单 | LLM + 营养规则 |
| 商品推荐 | 餐单 + 画像 + 记忆 + 商品标签 | Top N 商品 | 双塔 + 规则重排 |
| 营销策略 | 画像 + 生命周期 + 渠道偏好 | 触达时机 + 文案 + 优惠 | UCB / 上下文 bandit |
| 健康安全约束 | 禁忌 + facts + 推荐 | 拦截 / 改写 / 告警 | 规则引擎 |
| 家庭协同决策 | 多成员画像 + 共享餐 | 一桌菜方案 | LLM + 约束求解 |

### 5.2 决策优先级（贯穿全链路）

```
健康安全 / 禁忌
  > 健康事实（报告）
    > 个人健康标签
      > 设备近期状态
        > 当前问题
          > 行为记忆偏好
            > 营销转化
```

记忆和营销永远不能让步于健康红线。

## 6. Agent 编排层（L5）— 智能交互中枢

核心命题：**Agent 从"单点对话"升级为"家庭健康管家 + 集团营销助理"双角色协作。**

### 6.1 Agent 角色矩阵

| 角色 | 职责 | 工具集 |
| --- | --- | --- |
| 健康管家 | 报告解读、餐单、提醒、问答 | report_qa、profile_query、meal_plan、device_query |
| 家庭协调员 | 多成员日程协调、家庭聚餐方案 | family_calendar、shared_meal_plan |
| 营养师 | 阶段性食谱、慢病食养建议 | condition_meal_plan、expert_kb |
| 营销助理 | 商品推荐、券发放、加购引导 | mall_recommend、coupon、cart、order_track |
| 风控审核 | 健康安全 / 合规审核 | safety_check、compliance_check |
| 集团运营 | 集团活动配置、用户分群、营销策略 | campaign_config、segment_builder |

### 6.2 工具市场（Tool Marketplace）

未来扩展时，所有 Agent 工具走统一注册中心，支持：

- 工具定义（输入 / 输出 / 权限 / SLA）
- 工具评测（成功率 / 用户反馈 / 风险事件）
- 工具版本与灰度
- 工具上下线审批

### 6.3 多模态交互

- 文本（对话）
- 语音（智能音箱、车机）
- 图像（拍一拍冰箱、餐桌、药盒）
- 富交互（餐单卡片、报告解读长图、商品对比卡）

## 7. 转化与履约层（L6）— 从洞察到商品

核心命题：**让健康洞察真正转化为集团商品销售，并形成履约与复购闭环。**

### 7.1 商品中心

- **集团 SKU 全量接入**：福临门、长城、中茶、悦鲜活、香雪面粉、五谷道场、中粮我买网自营、蒙牛联合 SKU。
- **健康标签体系**：
  - 营养维度：低钠、低脂、低糖、高纤、高钙、高蛋白、无添加
  - 健康角色：控压主食、控糖零食、补钙乳品、降脂调味、儿童成长
  - 家庭角色：早餐 / 午餐 / 晚餐 / 共享餐桌 / 节庆礼盒
- **动态标签**：基于用户季节、地域、家庭结构的实时优选。

### 7.2 转化触点

| 触点 | 场景 | 形态 |
| --- | --- | --- |
| Agent 对话 | "爸爸今晚吃什么" | 餐单 + 推荐商品卡片 |
| 报告解读末梢 | 体检报告异常项 | "建议补充 X 类食材" → 商品卡 |
| 餐单页 | 一周餐单底部 | 食材一键加购 |
| 商城首页 | 健康画像驱动 | 个性化货架 |
| 主动提醒 | 复购临界点 / 季节变换 | Push / 短信 / 企业微信 |
| 处方化推送 | 慢病用户 | "营养师推荐方案" |
| 集团活动页 | 节庆 / 季节大促 | 健康主题活动 |
| 线下门店 | 大悦城 / 集团线下 | 扫码看健康建议 + 加购 |

### 7.3 履约与售后

- 一键加购、智能凑单（凑满减、凑免运）。
- 履约时效感知（结合冰箱库存提示补货）。
- 健康售后（吃后反馈、不适登记）。

## 8. 反馈闭环层（L7 下沉）— 持续优化与价值反哺

核心命题：**让每一次交互都反哺到数据资产和集团决策，形成"用户越用越聪明，集团越卖越精准"的飞轮。**

### 8.1 用户侧闭环

```
推荐 → 展示理由 → 用户反馈（采纳 / 拒绝 / 修改）
   → 写入 behavior_memory
   → 触发下一轮画像更新
```

### 8.2 健康效果闭环

```
干预（餐单 / 商品 / 提醒）
   → 设备数据 / 复测数据 / 下次报告
     → 健康时间线更新
       → 健康改善归因报告
         → 用户价值感知 → 粘性提升
```

### 8.3 营销转化闭环

```
曝光 → 点击 → 加购 → 下单 → 复购
   → 营销策略模型更新（bandit 反馈）
   → 集团选品 / 库存调整
```

### 8.4 集团价值反哺

| 反哺方向 | 数据来源 | 价值 |
| --- | --- | --- |
| 区域选品 | 区域健康画像 + 销量 | 区域差异化上架 |
| 新品研发 | 健康缺口 + 用户反馈 | 集团研发选题依据 |
| 供应链计划 | 健康趋势预测 | 种植 / 加工计划前置 |
| 品牌定位 | 用户对商品的健康角色认知 | 营销话术优化 |
| 集团会员运营 | 健康生命周期 | 精准分群营销 |
| 政策与 ESG | 健康改善统计 | 健康中国 / 乡村振兴叙事 |

## 11. 合规、安全与信任（贯穿全层）

### 11.1 数据合规

- 个人信息保护法、数据安全法、网络安全法、医疗机构管理条例合规评估。
- 家庭成员数据分别授权、分别可见。
- 跨境 / 跨集团数据走隐私计算或差分隐私。

### 11.2 健康合规

- 不做医学诊断，不替代医生。
- 所有推荐附"非医疗建议"提示。
- 涉及处方 / 慢病方案需专业营养师 / 医生审核。
- 健康免责声明与申诉渠道明示。

### 11.3 算法可解释

- 每个推荐必须展示依据链。
- 用户可点开"为什么推荐我"看完整链路。
- 算法变更需可灰度、可回滚。

## 12. 商业模式

### 12.1 C 端增值服务

- 免费层：基础报告解读 + 餐单 + 推荐。
- 会员层：家庭多成员 + 深度报告解读 + 营养师咨询。

### 12.2 B 端赋能

- 家庭健康趋势报告（脱敏后）。

## 16. 一页纸总结

> **粮达健康 3 年规划 = 一条主线 + 五个引擎 + 一个生态。**
>
> - 一条主线：采集 → 沉淀 → 理解 → 转化 → 反馈
> - 五个引擎：数据采集 / 数据资产 / 智能决策 / Agent 编排 / 转化履约
> - 一个生态：中粮集团消费品矩阵 + B 端供应链 + C 端家庭 + 第三方健康机构

落地节奏：

- **第一年**：补齐闭环 + 集团核心品类接入。
- **第二年**：沉淀数据资产 + 集团全品类 + 多端触点。
- **第三年**：生态化与 B 端赋能 + 商业化成熟。

最终目标：**让粮达健康成为中粮集团面向家庭的健康数据入口与精准营销引擎，成为健康中国战略下的央企标杆。**
