# 粮达健康技术方案阐述

粮达健康定位为“基于健康报告理解、家庭健康画像与 Agent 编排的家庭健康智能营销系统”。系统采用前后端分离架构：前端基于 React + TypeScript 构建多页面业务界面，后端基于 FastAPI 提供 Agent、报告、会员、设备、商城等服务，底层使用 MySQL 管理业务数据，Milvus 承载向量检索，mem0 负责长期互动记忆。

在数据链路上，系统以体检报告为核心入口。用户上传 PDF 后，后端完成文本提取与 OCR 补偿、分块切片、向量化入库，并保留页码与原文片段，支持后续证据追溯。同时通过 LLM 抽取异常指标、风险与建议，形成结构化健康事实。系统再结合家庭成员档案、近 7 天设备指标和历史互动记忆，聚合为可计算的健康画像，输出长期风险、近期状态、饮食原则与个性化偏好。

在决策链路上，LangChain Agent 不直接依赖单轮 prompt，而是按意图调用报告检索、记忆检索、餐单生成和商品推荐等工具，实现“理解问题-调用能力-生成答案-给出推荐”的闭环。对于每次回答，系统会同步展示生成依据与推荐依据，确保用户能看到回答来自哪份报告、哪段记忆和哪类商品标签，提升可信度与可解释性。

本方案的创新点不在单一问答，而在于把“报告理解、健康画像、个性化建议、商品转化”串成一条完整业务链路，使 AI 同时具备健康理解能力、家庭场景适配能力和营销转化能力，适合健康管理与精准推荐场景落地。
